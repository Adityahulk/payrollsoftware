================================================================================
          MICROTECHNIQUE PAYROLL & HR MANAGEMENT PLATFORM - ARCHITECTURE REFERENCE
================================================================================

This document provides a detailed overview of the system architecture, database
tables, columns, user roles, API routing, and core functionalities of the 
Microtechnique Payroll and HR system.

--------------------------------------------------------------------------------
1. TECHNOLOGY STACK OVERVIEW
--------------------------------------------------------------------------------
- Backend: ASP.NET Core Web API (.NET 8.0)
  * Database Connector: Npgsql (PostgreSQL client)
  * Data Mapping: Dapper (configured for snake_case conversion)
  * Authentication: JWT Bearer tokens
  * Real-Time Alerts: ASP.NET Core SignalR (WebSockets)
  * Email Service: Resend API integration for OTP & notifications
  * Task Scheduler: MonthEndHostedService (automates payroll checks)

- Frontend: React.js SPA (configured in react-crud)
  * Styling: Vanilla CSS / Modern Tailwind CSS
  * State & Cache Management: TanStack React Query (v4/v5)
  * Notifications: React Hot Toast
  * Routing: React Router DOM (v6)

--------------------------------------------------------------------------------
2. DATABASE SCHEMA (TABLES & COLUMNS)
--------------------------------------------------------------------------------
All tables are mapped to PostgreSQL with Dapper handling translation between 
C# PascalCase models and database snake_case columns.

[1] Table: t_users
---------------------------------------------
Stores all user information, credentials, and payment profiles.
- empid (INTEGER - SERIAL - PRIMARY KEY)
- spaceid (INTEGER - Nullable FK)
- name (VARCHAR(200) - Nullable)
- email (VARCHAR(255) - UNIQUE)
- passwordhash (VARCHAR(255))
- gender (VARCHAR(50))
- status (VARCHAR(50) - e.g., 'Active', 'Pending', 'Inactive')
- role (VARCHAR(50) - 'SuperAdmin', 'Admin', 'Manager', 'TeamLead', 'Employee')
- phone (VARCHAR(50))
- address (TEXT)
- dateofjoining (DATE - Default: CURRENT_DATE)
- profilephotourl (TEXT)
- backupemail (VARCHAR(100))
- accountnumber (VARCHAR(50))
- bankname (VARCHAR(100))
- accountholdername (VARCHAR(150))
- ifsccode (VARCHAR(20))
- upiid (VARCHAR(100))
- is_approved (BOOLEAN - Default: FALSE)
- statusreason (TEXT)
- statusbysuperadmin (BOOLEAN - Default: FALSE)

[2] Table: t_spaces
---------------------------------------------
Defines organizational spaces (departments, client accounts, or contract units).
- spaceid (INTEGER - SERIAL - PRIMARY KEY)
- spacename (VARCHAR(255))
- adminid (INTEGER - FK to t_users.empid)
- numberofemployees (INTEGER)
- createdat (TIMESTAMP WITHOUT TIME ZONE)
- isactive (BOOLEAN - Default: TRUE)
- numberofbreaks (INTEGER - Default: 1)
- breaktime (INTEGER - break limit in minutes, e.g., 60)
- workstarttime (TIME - Standard clock-in limit)
- workendtime (TIME - Standard clock-out limit)
- workinghours (INTEGER - Required daily work duration)
- type (VARCHAR(20) - 'Department' or 'Contract')
- enddate (DATE - Nullable, used for Contract spaces)
- workingdays (TEXT - JSON array string containing short day names, e.g., '["Mon","Tue","Wed","Thu","Fri"]')
- maxspaces (INTEGER - Limit for spaces/subs)

[3] Table: t_attendance
---------------------------------------------
Logs daily working and checking activities.
- attendanceid (INTEGER - SERIAL - PRIMARY KEY)
- empid (INTEGER - FK to t_users.empid)
- attendancedate (DATE)
- clockin (TIMESTAMP)
- clockout (TIMESTAMP)
- totalhours (NUMERIC(5,2))
- status (VARCHAR(50) - e.g., 'Present', 'Late', 'Absent')
- lateminutes (INTEGER)
- earlyexitminutes (INTEGER)
- breakhours (NUMERIC(5,2) - Default: 0)
- overtimehours (NUMERIC(5,2) - Default: 0)
- createdat (TIMESTAMP)

[4] Table: employeebreaks
---------------------------------------------
Tracks individual break durations to enforce daily thresholds.
- breakid (INTEGER - SERIAL - PRIMARY KEY)
- empid (INTEGER)
- breakstart (TIMESTAMP WITHOUT TIME ZONE)
- breakend (TIMESTAMP WITHOUT TIME ZONE)
- totalbreakminutes (INTEGER - Default: 0)
- createdat (TIMESTAMP WITHOUT TIME ZONE)

[5] Table: t_leaves
---------------------------------------------
Tracks personal/medical days off requested by employees.
- leaveid (INTEGER - SERIAL - PRIMARY KEY)
- empid (INTEGER - FK to t_users.empid)
- spaceid (INTEGER)
- leavedate (DATE)
- reason (TEXT)
- status (VARCHAR(50) - 'Pending', 'Approved', 'Rejected')
- leavetype (VARCHAR(20) - 'Normal', 'Emergency', 'College')
- halfday (BOOLEAN - Default: FALSE)
- approvedby (INTEGER - FK to t_users.empid)
- createdat (TIMESTAMP)
* CONSTRAINT: uq_emp_leavedate (UNIQUE empid + leavedate)

[6] Table: t_space_leave_config
---------------------------------------------
Defines monthly leave quotas for a specific Space.
- configid (INTEGER - SERIAL - PRIMARY KEY)
- spaceid (INTEGER - UNIQUE FK)

[7] Table: t_worklogs

[8] Table: t_projects

3. USER ROLES & PERMISSION SUMMARY
--------------------------------------------------------------------------------
The application utilizes JWT roles. The route permissions are mapped as follows:

- SuperAdmin:
  * Manages Admin approvals and profile lockdowns (is_approved, statusbysuperadmin).
  * Manages global settings and active directory of organizations/spaces.
  * Pages: /superadmin (controls SuperAdminDashboard)

- Admin:
  * Owns one or multiple Space structures.
  * Manages allowances, standard deductions, and basic salaries.
  * Adds and registers employees, managers, and team leads to their space.
  * Processes monthly payroll and issues direct transfer payouts.
  * Reviews leaves, updates configs, issues warnings, posts announcements.
  * Pages: /admin, /admin/users, /admin/projects, /admin/attendance, 
           /admin/salary, /admin/queries, /admin/spaces, /admin/leaves, 
           /payroll/:spaceId, /admin/profile

- Manager:
  * Direct manager of the space / department.
  * Reviews leave applications, tracks attendance sheets, monitors user logs.
  * Registers/updates employees in the department (/employee/manage-users).
  * Pages: /employee, /employee/attendance, /employee/salary, /employee/queries, 
           /employee/worklogs, /employee/manage-users, /employee/all-employees, 
           /employee/progress, /employee/projects, /employee/profile, 
           /employee/leaves

- TeamLead:
  * Project supervisor.
  * Creates Projects, writes deliverables, and assigns Tasks to employees.
  * Approves completed tasks, reviews worklog lists, and tracks project progress.
  * Pages: /employee (mapped to TLDashboard), /employee/projects, 
           /employee/attendance, /employee/leaves (read/approve limits)

- Employee:
  * Standard workspace contributor.
  * Clocks in and out daily. Records break sessions (within daily limits).
  * Submits worklogs detailing hours worked against assigned tasks.
  * Applies for leaves (Normal, Emergency, College) and monitors status.
  * Submits queries to admins and reads notice board announcements.
  * Views monthly payslips, CTC summaries, and annual stats.
  * Pages: /employee, /employee/attendance, /employee/salary, /employee/queries,
           /employee/worklogs, /employee/all-employees, /employee/progress, 
           /employee/projects, /employee/profile, /employee/leaves

--------------------------------------------------------------------------------
4. SYSTEM MODULES & FUNCTIONS DESCRIPTION
--------------------------------------------------------------------------------

[A] AUTHENTICATION & SECURITY
- Registration & Onboarding: Users register under specific roles. Admin registers 
  managers/employees to their specific department/space.
- Approvals: Admin accounts must be approved by SuperAdmin before logging in.
- OTP System: Forgotten passwords generate verification pins stored in t_otp.
  The service sends a password reset email using the Resend provider.

[B] TIME & ATTENDANCE TRACKING
- Clock-In: Saves clock-in timestamp. Checks department standard WorkStartTime.
  If the employee logs in late, late minutes are calculated:
     LateMinutes = ClockIn - StandardStartTime
  and attendance status is set to "Late".
- Breaks: Employees can pause daily tasks. It writes to employeebreaks.
  Total break minutes are computed and saved.
- Clock-Out: Validates exit time. Checks standard exit time and total working
  hours required (e.g. 8 hours). Shortfalls calculate EarlyExitMinutes.
- Daily Status: Mapped automatically as "Present", "Late", or "Absent".

[C] DYNAMIC PENALTY SYSTEM (PenaltyCalibrator)
- Automated calculation of attendance-based salary adjustments:
  * LATE ARRIVAL PENALTY: Deducts salary if the employee is late more than a
    configured grace threshold.
  * ABSENT PENALTY: Deducts salary for workdays with no attendance logs or unapproved
    leaves.
  * EARLY EXIT PENALTY: Fine calculations mapped to early exit minutes.

[D] LEAVE MANAGEMENT
- Leaves are categorized into Normal, Emergency, and College.
- Employees check their remaining leaf allocations (remaining balance calculated
  dynamically from space leave policies).
- Submitting a request updates t_leaves as "Pending".
- SignalR triggers an instant push to Admin/Manager devices.
- Once updated to "Approved" or "Rejected", the audit record is saved.

[E] PROJECT & TASK COLLABORATION
- TeamLeads create projects and tasks with fields like priority and start dates.
- Employees can view their assigned task lists and mark them as InProgress or Completed.
- Task completion broadcasts real-time alerts to the TL and Admin.

[F] WORKLOG SUBMISSIONS
- Employees submit daily details about task hours worked.
- System evaluates actual hours vs estimated hours to draw completion rates.

[G] NOTICE BOARD & HELPDESK
- Notice Creation: Broadcasts corporate bulletins.
- Ticket Support Queries: Employees generate tickets under "Query" type.
  Admins/Managers receive alerts, provide replies (repliedby, reply), and
  toggle status (Open/Closed).

[H] SALARY & PAYROLL CALCULATIONS
- Structure: Net pay calculations take base salary, add active fixed/percentage
  allowances (t_allowances), and subtract deductions (t_deductions) and late/absenteeism
  penalties calculated dynamically.
- Process Payroll: Admin processes month-end payroll, creating transaction logs in
  t_payrollpayments, and archiving finalized payslip details in t_payslips (with
  breakdowns and active bank details at the time of payment).
- Auto-alerts: MonthEndHostedService triggers automatically on the last day of the
  month to notify Admins to verify payroll components.
